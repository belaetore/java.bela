No episódio de hoje, nós vamos aprender um pouco mais sobre POO (Pilares da Orientação a Obejtos) de uma forma fácil de se compreender e com direito a exemplos e explicações sobre os pilares em Java!

__________________________________________________________

A orientação a objetos é uma forma de interação entre unidades chamadas de objetos. Os pilares podem ser definidos em quatro tipos: herança, encapsulamento, abstração e polimorfismo.

__________________________________________________________

ENCAPSULAMENTO

O encapsulamento serve para dar mais segurança a sua programação, permitindo que os detalhes de implementação de um objeto sejam alterados sem afetar o restante do sistema, tornando o código mais fácil de manter. Além disso, o encapsulamento pode ser usado para proteger métodos e dados de uma classe de acesso indevido, evitando a violação de dados confidenciais.



__________________________________________________________

Neste pequeno cenário, temos uma variável representando uma idade e posteriormente duas funções que leem/escrevem nela. Temos então uma classe Aluno, agrupando as variáveis e funções que tínhamos anteriormente.

__________________________________________________________


HERANÇA

Herança é um princípio que permite criar novas classes com base em classes existentes. A classe herdada é chamada de classe base ou superclasse, e a nova classe criada é chamada de classe derivada ou subclasse. A herança permite a reutilização de código, facilita a criação de hierarquias de classes e aumenta a escalabilidade do sistema.



__________________________________________________________

Nesse exemplo, podemos ver que a CLASSE pessoa possui alguns atributos como nome, CPF e data de nascimento. Depois é criado um OBJETO pessoa, usado para receber os dados, para preencher os atributos fazendo com que o programa forneça os dados completos.

__________________________________________________________


ABSTRAÇÃO

A abstração, é o processo de ocultar os detalhes complexos e focar apenas nas características essenciais de um objeto ou conceito. Isso permite que o programador se concentre na interação com o objeto em um nível mais alto, sem se preocupar com a implementação interna.

__________________________________________________________

A classe abstrata Animal define um método “fazerSom()” sem implementação. A subclasse Cachorro implementa esse método, imprimindo "O cachorro late." Na classe Main, um objeto Cachorro chama “fazerSom()”, mostrando a mensagem.

__________________________________________________________

POLIMORFISMO

Polimorfismo é quando um método pode ter diferentes comportamentos dependendo do tipo de objeto que o utiliza. Em outras palavras, ele permite que a mesma ação seja realizada de maneiras variadas, conforme o tipo específico de objeto envolvido.

__________________________________________________________

O código demonstra polimorfismo em Java com a classe base Carro e suas subclasses CarroEsportivo e CarroSUV. A classe Carro tem um método ligar() que imprime uma mensagem genérica. As subclasses substituem esse método para oferecer mensagens específicas: CarroEsportivo imprime "O carro esportivo está ligado. Vroom vroom!" e CarroSUV imprime "O SUV está ligado. Pronto para a aventura!". No método main, mesmo com referências do tipo Carro, as chamadas a ligar() executam as versões sobrescritas conforme o tipo real do objeto. Isso ilustra como o polimorfismo permite comportamentos diferentes a partir de uma interface comum.
